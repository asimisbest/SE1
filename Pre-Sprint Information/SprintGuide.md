# Sprint Process

## Sprint Flow

### Sprint Planning
- Review product backlog
- Select high-priority items
- Break items into smaller tasks
- Estimate work
- Move tasks into sprint backlog

### During Sprint
- Work on assigned tasks
- Update sprint board
- Commit code frequently
- Open pull requests
- Fix issues from testing/review

### Daily Check-in
- What was completed?
- What is next?
- Any blockers?

### Sprint Review / Demo
- Show working features
- Demonstrate real functionality
- Gather feedback

### Retrospective
- What went well?
- What didn’t?
- What will we improve next sprint?
