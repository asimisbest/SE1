# Design Standards

## Principles
- Keep designs lightweight
- Only document when it reduces confusion
- Focus on clarity over formality

## Artifacts We Use
- Use Case Diagrams
- UML Class Diagrams
- Sequence Diagrams
- Wireframes / Mockups
- ER Diagrams (if needed)
- API / Interface Documentation

## Rule
Create design documentation when:
- Feature is complex
- Multiple systems interact
- Team alignment is needed
